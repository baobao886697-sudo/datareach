# 积分充值页面审查 /recharge

## 页面结构
- 顶部：USDT-TRC20 标签 + "积分充值" 标题 + "使用USDT充值积分，1 USDT = 100 积分"
- 充值优惠活动横幅：100+ USDT +2%, 200+ +4%, 500+ +8%, 1000+ +10%
- 右侧：QR码 + "创建订单开始充值" + 充值流程3步
- 充值积分区域：当前余额 99,995,035 积分
- 4个金额选项：50/100/200/500 USDT（100+有赠送）
- 自定义充值金额输入框（最低50 USDT）
- 费用计算：基础积分、实际获得、兑换比例、应付金额
- "创建充值订单"按钮（橙色渐变，醒目）
- 充值记录区域：显示"暂无充值记录"

## 问题发现

### 设计问题
1. 50 USDT 选项没有赠送标签，其他3个有 - 一致性可以更好，可以标注"无赠送"或不标注
2. "应付金额 ~50 USDT" 前面的 ~ 符号 - 用户可能困惑为什么不是精确金额
3. "实际金额将包含唯一尾数以便自动识别" - 这个说明很重要但字体太小
4. QR码区域在未创建订单时显示占位图 - 合理

### 文案问题
5. "充得越多，送得越多！" - 口语化，可以更专业
6. "*最高可享15%赠送，详情请查看完整优惠规则" - 但没有链接可以查看完整规则
7. "自定义充值金额（最低 50 U..." - 文字被截断了

### 功能问题
8. 充值记录显示"暂无充值记录" - 但该用户有大量积分，可能是测试账号直接加的
9. 没有充值历史的搜索/筛选功能
10. 没有积分消费明细/账单功能

### UI 问题
11. 充值流程3步说明在右侧 - 位置合理
12. "创建充值订单"按钮橙色渐变很醒目 ✓
13. 金额选项卡片选中状态（50 USDT 有绿色边框）清晰 ✓
