# 铃铛红点和已读测试结果

## 测试流程

1. 管理员发送一条个人消息给用户
2. 刷新用户仪表盘 -> 铃铛显示红色数字"1"
3. 点击铃铛打开通知面板 -> 显示未读消息（积分到账通知），有蓝色未读圆点
4. 点击未读消息 -> 消息标记为已读
5. 铃铛红色数字消失 -> 恢复正常状态

## 功能正常的部分

1. 未读消息计数正确显示在铃铛上
2. 点击消息可以标记已读
3. 已读后铃铛恢复正常（无红点）
4. "全部已读"按钮在有未读消息时显示
5. 公告展开/收起功能正常
6. 公告类型标签（紧急/警告）在用户端正确显示

## 需要修复的问题

### 问题1: 铃铛红点没有闪烁动画
当前红点是静态的 bg-red-500，没有任何动画效果。用户要求"红色闪烁明显的标记信息"。
需要添加 CSS pulse/blink 动画。

### 问题2: 通知面板 ScrollArea 高度可能不够
当公告展开后加上消息列表，内容可能超出 max-h-[400px]，需要确认滚动是否正常。

### 问题3: 公告没有显示类型标签
在通知面板中，公告只在 warning 和 error 类型时显示标签，info 和 success 类型不显示标签。应该所有类型都显示。
