# 搜索平台页面审查

## TPS (TruePeopleSearch) /tps
- 页面结构完整：标题+特色标签+搜索模式+姓名输入+高级选项+积分余额+预扣费用+核心优势+快速入门+费用说明
- 特色标签：推荐数据源、房产价格、最新号码
- 搜索模式：仅姓名搜索 / 姓名+地点
- 预扣费用说明清晰：搜索页25页×0.3=7.5，详情页200条×0.3=60.0，总额67.5积分
- "多扣少补：任务完成后退还多余积分" - 好的说明
- 费用说明：每页0.3积分，每条详情0.3积分，缓存命中免费，缓存180天

## SPF (SearchPeopleFree) /spf/search
- 页面结构与 TPS 一致，布局统一
- 特色标签：独家数据、婚姻状态、电子邮件
- 特色功能卡片：电子邮件、婚姻状态、就业状态、数据确认日期
- 费用预估：搜索任务数0个，单次搜索费用0.85+0.85=1.7积分
- 费用说明：每页0.85积分，每条详情0.85积分

## Anywho /anywho
- 页面结构与 TPS/SPF 一致
- 特色标签：AT&T 官方、婚情状况、运营商信息
- 特色功能卡片：AT&T 官方、婚姻状况、运营商信息、电话类型
- 费用预估(混合模式)：搜索任务数0个，年龄段数量2个
- 费用说明：每页0.5积分，每条详情0.5积分

## 发现的问题汇总

### 一致性问题
1. 三个搜索页面(TPS/SPF/Anywho)布局一致性好 ✓
2. textarea 中的 placeholder 都是 "John Smith\nJane Doe\nRobert Johnson" - 一致 ✓
3. 费用计算方式不同(TPS用预扣，SPF用预估，Anywho用混合模式) - 可能让用户困惑

### 文案问题
4. "已输入 0 个姓名" - textarea 有 placeholder 但显示0个，因为 placeholder 不算输入
5. Anywho "婚姻状况" vs SPF "婚姻状态" - 用词不一致
6. PeopleSearchNow "已有 1,234 人关注" - 看起来像假数据
7. PeopleSearchNow "预计上线：2026年Q1" - 已经快过期了

### 功能问题
8. 404页面 /truepeoplesearch 不存在 - 但这不是侧边栏的链接
9. 404 页面是英文的 "Page Not Found" - 应该是中文
10. "高级选项" 折叠区域 - 需要展开查看具体内容
11. 搜索历史按钮在右上角 - 位置合理

### UI 问题
12. 所有搜索页面左侧内容区域大量空白（textarea下方到底部）
13. 右侧积分余额和费用预估卡片位置合理
